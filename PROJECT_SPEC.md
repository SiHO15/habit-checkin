# 个人每日状态打卡软件需求文档

## 项目目标

做一个自己使用的本地打卡软件，运行在浏览器中，不需要登录，不需要后端，不需要数据库。数据保存在浏览器 localStorage 中。

## 技术栈

- React
- Vite
- CSS
- localStorage

## 第一版功能

### 1. 日期

用户可以记录某一天的状态。

默认日期为当天日期，格式为：

YYYY-MM-DD

用户也可以手动修改日期，用于补记之前某一天的状态。

### 2. 睡眠时间记录

用户可以输入当天睡眠时间。

要求：

- 使用数字输入框
- 单位为小时
- 可以输入小数，例如 7.5

睡眠时间需要有一个附录功能。

睡眠附录要求：

- 默认可以为空
- 使用多行文本框
- 用户可以自由填写睡眠情况补充
- 例如：昨晚入睡较晚，中途醒了一次

### 3. 有氧运动记录

用户可以选择当天是否做了有氧运动。

要求：

- 使用“是 / 否”选择
- 默认值可以为“否”

有氧运动需要有一个附录功能。

有氧附录要求：

- 默认可以为空
- 使用多行文本框
- 用户可以自由填写有氧运动补充
- 例如：慢跑 30 分钟，强度中等

### 4. 力量训练记录

用户可以选择当天是否进行了力量训练。

要求：

- 使用“是 / 否”选择
- 默认值可以为“否”

力量训练需要有一个附录功能。

力量训练附录要求：

- 默认可以为空
- 使用多行文本框
- 用户可以自由填写力量训练补充
- 例如：练了胸和三头，整体状态不错

### 5. 饮食记录

用户可以输入当天吃了几顿饭。

要求：

- 使用数字输入框
- 单位为顿
- 一般输入 0、1、2、3、4 等数字

饮食记录需要有一个附录功能。

饮食附录要求：

- 默认可以为空
- 使用多行文本框
- 用户可以自由填写饮食情况补充
- 例如：早餐没吃，午饭和晚饭正常

### 6. 额外补充

页面需要一个独立的额外补充区域。

要求：

- 使用多行文本框
- 用户可以自由记录除了睡眠、有氧、力量训练、饮食以外的其他内容
- 可以为空

### 7. 保存功能

页面需要一个“保存今日记录”按钮。

点击后：

- 将当前日期的数据保存到 localStorage
- 如果该日期已经存在记录，则覆盖旧记录
- 保存成功后给用户一个明显提示

### 8. 历史记录功能

页面下方需要显示历史记录列表。

每条历史记录需要显示：

- 日期
- 睡眠时间
- 是否做了有氧运动
- 是否进行了力量训练
- 吃了几顿饭

每条历史记录需要支持：

- 展开查看详情
- 删除该条记录
- 点击某条记录后，可以把该记录加载回上方表单进行修改

### 9. 页面风格

页面要求：

- 简洁
- 干净
- 卡片式布局
- 适合长期自用
- 输入框、按钮、文本框要有基础美化
- 手机和电脑上都能正常显示

## 数据结构建议

每条记录可以使用类似结构：

{
  date: "2026-05-15",
  sleepHours: 7.5,
  sleepNote: "昨晚睡得比较晚",
  cardio: true,
  cardioNote: "慢跑 30 分钟",
  strength: false,
  strengthNote: "",
  meals: 3,
  mealsNote: "三餐正常",
  extraNote: "今天整体状态不错"
}
